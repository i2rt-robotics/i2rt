# v1.1 Release Notes

**Date:** 2026-05-05

## Highlights

This release moves robot configuration to YAML, adds a new `flexible_4310` soft-tip gripper, ships a browser-based teleop interface alongside an overhauled MuJoCo control loop, and brings reliability and ergonomic improvements to the motor driver and `minimum_gello` example.

## YAML-Driven Robot Configuration

- **Per-arm and per-gripper YAML configs** under `i2rt/robots/config/*.yml` describe motor lists, directions, PD gains, and gravity-comp factors. The hardcoded `_ArmHWConfig` / `_GripperHWConfig` dataclasses have been retired.
- **Joint limits read from MJCF**: `get_yam_robot()` now extracts joint limits directly from the arm's MuJoCo XML, making the model the single source of truth.
- **Centralised model paths** in `i2rt/robot_models/__init__.py` — every arm and gripper XML is reachable via a named constant.

## New Gripper: `flexible_4310`

- Added `i2rt/robot_models/gripper/flexible_4310/` (XML, URDF, assets) — a DM4310-driven gripper with flexible soft tips, selectable via `GripperType.FLEXIBLE_4310`.

## Interactive Visualization & Teleop

- **`i2rt.utils.viser_control_interface`** — new browser-based 3-D viewer with a read-only safety gate, three control modes (mirror, IK drag, per-joint sliders), teaching-handle SYNC/RECORD button indicators, and a normalised 0–1 gripper slider that matches the robot API.
- **`i2rt.utils.mujoco_control_interface`** overhauled — threaded control loop, per-joint sliders, self-collision detection that blocks unsafe commands, gravity-comp integration, gripper-index support, and a logging mode.
- **Interactive 3-D robot viewer** added to the product docs for YAM and BIG YAM.
- **`examples/control_with_viser/`** added; **`examples/control_with_mujoco/`** updated to use the new helpers (now accepts `--log`).

## Motor Driver Reliability

- CAN bandwidth check and per-driver rate recording in `dm_driver.py`; communication frequency exposed via the new `comm_freq` property.
- Driver lock switched from `Lock` to `RLock`; duplicate control-loop thread starts are now guarded.
- `CanInterface` records its channel name and drains stale frames (`_drain_bus()`) before motor bring-up; the same-bus device lock is held during driver init.
- CAN receive timeout reduced for faster control-loop turnaround.

## SDK & API

- `get_yam_robot()` accepts new keyword arguments: `gravity_comp_factor`, `gripper_limits_override`, `gripper_kp`, `gripper_kd` for per-call gripper and gravity tuning.
- `ArmType.NO_ARM` selects a gripper-only robot via `get_yam_robot()`.
- `MotorChainRobot` adds teaching-handle support, joint log mode, and torque logging.

## Examples

- **`minimum_gello.py` moved** from `scripts/` to `examples/minimum_gello/` with a dedicated README. New CLI flags: `--ee_mass`, `--arm`, `--sim`. Visualizer fixes — normalised gripper slider.
- **Motor configuration tools** (`set_zero.py`, `ping_motors.py`, `set_timeout.py`) now default to **all motors 1–7** when `--motor_id` is omitted, matching one another.

## Dependencies

- `pyyaml>=6.0.0` added (required for YAML-driven config).
- `mujoco` version pin loosened.

## Dev / CI

- Unit-test workflow migrated to `astral-sh/setup-uv` with `uv sync --dev`.
- `ruff`, `pyright`, and `pre-commit` added as dev dependencies.
